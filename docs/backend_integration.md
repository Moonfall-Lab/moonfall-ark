# Moonfall Runtime Backend Integration

This branch implements the MVP IR contract from `前后端对接文档 · Moonfall Runtime (MVP IR).md`.

## Base URLs

- Local REST: `http://127.0.0.1:8000/api`
- Local WebSocket: `ws://127.0.0.1:8000/ws`
- LAN use: replace `127.0.0.1` with the backend machine IPv4 address.

## Docker Unified Environment

The `back-end` branch includes a Docker runtime:

```bash
docker compose up --build
```

After startup:

- REST: `http://127.0.0.1:8000/api`
- WebSocket: `ws://127.0.0.1:8000/ws`
- Health: `http://127.0.0.1:8000/api/health`

See `docs/docker.md` for environment variables, volume behavior, and shutdown commands.

## WebSocket Envelope

All WebSocket messages use one envelope:

```json
{
  "topic": "input.card",
  "source": "frontend",
  "timestamp": 1720000000.0,
  "payload": {
    "player_id": "p1",
    "card_id": "collect_priority"
  }
}
```

## Server Broadcast Topics

- `state.world`: full MVP IR world payload.
- `state.event`: one discrete event for timeline or marquee display.
- `cmd.robot`: robot command generated from voice input.
- `cmd.arm`: arm command generated by debug/director paths.
- `cmd.humanoid`: reserved by the contract.

## Client Input Topics

- `sensor.hr`: `{ "player_id": "p1", "heart_rate": 105 }`
- `input.voice`: `{ "player_id": "p1", "text": "一号车去中央采集" }`
- `input.card`: `{ "player_id": "p1", "card_id": "collect_priority" }`
- `input.declare_launch`: `{ "player_id": "p1" }`

## REST Methods

| Method | Path | Purpose |
|---|---|---|
| `GET` | `/api/health` | Health check. |
| `GET` | `/api/state` | Returns the `state.world.payload` object. |
| `GET` | `/api/config` | Returns current game config IDs for frontend rendering. |
| `POST` | `/api/config/load` | Reloads config from `config_path` or inline `config`. |
| `POST` | `/api/control/start` | Starts the match and broadcasts `state.world`. |
| `POST` | `/api/control/reset` | Resets session state and broadcasts `state.world`. |
| `POST` | `/api/input/hr` | REST equivalent of `sensor.hr`. |
| `POST` | `/api/input/voice` | REST equivalent of `input.voice`; returns `cmd.robot`. |
| `POST` | `/api/input/card` | REST equivalent of `input.card`. |
| `POST` | `/api/input/declare_launch` | REST equivalent of `input.declare_launch`. |
| `POST` | `/api/debug/set_var` | Sets a scoped runtime variable. |
| `POST` | `/api/debug/trigger_event` | Emits a director event. |
| `GET` | `/api/logs/ai` | Returns recent AI log records. |
| `GET` | `/api/logs/functional` | Returns recent functional log records. |

## Useful cURL Examples

```bash
curl http://127.0.0.1:8000/api/config
curl http://127.0.0.1:8000/api/state
curl -X POST http://127.0.0.1:8000/api/input/declare_launch ^
  -H "Content-Type: application/json" ^
  -d "{\"player_id\":\"p1\"}"
curl -X POST http://127.0.0.1:8000/api/debug/set_var ^
  -H "Content-Type: application/json" ^
  -d "{\"scope\":\"faction\",\"id\":\"pa\",\"var\":\"fuel\",\"value\":5}"
```

## State Shape

`GET /api/state` and `state.world.payload` contain:

- `session_id`, `game_id`, `schema_version`
- `phase`, `turn`
- `global.moon_rage`, `global.moon_tier`
- `factions[]` with `id`, `players`, `rank`, `vars`
- `units[]` with `id`, `faction`, `kind`, `pose`, `status`, `carrying`
- `zones[]` with `id`, `kind`, `active`, `intensity`
- `rank_order`, `winner`
